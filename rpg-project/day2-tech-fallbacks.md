# day2-tech-fallbacks.md

**5个最常见失败及快速修复**

---

## F1: 事件不触发

**现象**: 走到事件位置没反应

**快速诊断**:
1. 检查事件页触发条件
2. 检查优先级设置
3. 检查开关/变量条件

**修复步骤**:
```
1. 右键事件 → 编辑
2. 确认"触发条件":
   - 自动执行: 用于开场
   - 确定键: 用于NPC对话
   - 玩家接触: 用于传送点
   - 并行处理: 用于自动检测
3. 确认"优先级":
   - NPC: "与人物相同"
   - 传送点: "低于人物"
4. 检查条件开关是否已满足
```

**验证**: 测试游戏重新触发

---

## F2: 传送错误/到不了目标地图

**现象**: 传送后黑屏/到错地图/卡住

**快速诊断**:
1. 检查场所移动命令
2. 检查目标地图是否存在
3. 检查目标坐标是否可通行

**修复步骤**:
```
1. 打开传送事件
2. 检查命令"场所移动":
   - 地图: 是否正确选择
   - X坐标: 1-地图宽度
   - Y坐标: 1-地图高度
3. 去目标地图检查:
   - 该坐标是否有tile
   - 该坐标是否可通行
4. 修正坐标后保存
```

**验证**: 测试传送

---

## F3: 必败战斗不触发/无限战斗

**现象**: 战斗一直进行不结束，或打赢了没败退

**快速诊断**:
1. 检查Troop Event设置
2. 检查脚本条件
3. 检查战斗处理命令

**修复步骤**:
```
1. 数据库 → Troop → 选择Troop 1
2. 检查Troop Event页面:
   - 触发: "回合结束"
   - 范围: "整个队伍"
3. 检查条件:
   - 条件类型: "脚本"
   - 脚本内容:
     $gameParty.members()[0].hp <= $gameParty.members()[0].mhp * 0.3 || BattleManager._turnCount >= 5
4. 检查事件命令:
   - "中断战斗"必须存在
   - 或调用Common Event
5. 保存Troop
```

**验证**: 测试战斗，5回合后或HP<30%自动结束

---

## F4: 败退后没给家谱/流程断

**现象**: 败退后卡死、没获得物品、没传送

**快速诊断**:
1. 检查Common Event 2
2. 检查增减物品命令
3. 检查传送命令

**修复步骤**:
```
1. 数据库 → 公共事件 → ID 2
2. 检查命令序列:
   1. 显示文本: "你支撑不住了..."
   2. 显示文本: "黑衣人撤退..."
   3. 增减物品: 选择"残破家谱", +1
   4. 开关操作: 12号 = ON
   5. 场所移动: M001, X:10, Y:10
3. 关键检查点:
   - 物品ID是否正确
   - 传送目标地图是否存在
   - 传送坐标是否有效
4. 保存公共事件
```

**验证**: 测试败退，检查物品栏和位置

---

## F5: 流程在某处断开/无法继续

**现象**: 做完A后无法触发B，流程卡住

**快速诊断**:
1. 检查开关传递
2. 检查事件页条件
3. 检查地图连接

**修复步骤**:
```
1. 确定断点位置（哪个事件后卡住）
2. 检查前置事件的结尾:
   - 是否设置了必要的开关？
   - 例如: 师尊任务后应设置S[11]=ON
3. 检查后续事件的开始:
   - 条件是否要求前置开关？
   - 例如: 传送点要求S[11]=ON
4. 检查地图连接:
   - 出口事件的目标地图是否正确？
   - 目标地图的入口坐标是否可通行？
5. 修正后保存
```

**验证**: 从断点前重新测试

## 通用调试技巧

### 快速定位问题

1. **使用测试游戏**: Alt+R 快速启动
2. **使用调试开关**: 创建一个开关"debug_mode"，用于跳过部分流程
3. **添加临时文本**: 在事件关键点添加"显示文本"，确认执行到哪一步
4. **检查独立开关**: 事件执行过一次后自开关A=ON，可能阻止再次触发

### 重置测试

```
需要重新测试时:
1. 游戏 → 返回标题
2. 或: 关闭测试窗口重新Alt+R
3. 注意: 数据库修改不需要重启，但事件修改需要重新进入地图
```

### 紧急回退

如果修改后问题更严重:
```
1. 关闭RPGMaker（不保存）
2. 从Git恢复之前的版本
3. 或: 使用备份的data文件夹
```

---

**记住**: 大部分问题是开关/条件设置错误，仔细检查事件页的条件部分。
