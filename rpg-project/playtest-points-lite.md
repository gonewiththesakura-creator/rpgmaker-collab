# Playtest Points Lite

**Purpose**: 5 key narrative test points for vertical slice demo validation
**Based on**: writer-training/week-04-final.md (Playtest design)
**Status**: Ready for Tech / QA
**Scope**: Ch1-Ch4 full playthrough

---

## 核心原则

- **每个测试点对应一个核心叙事目标**
- **验证方式 = 可量化的事件触发日志 + 玩家行为观察**
- **通过标准清晰**（百分比或行为阈值）
- **无需额外工具**，RPGMaker MZ 自带事件日志即可收集

---

## 测试点 TP-01: 谢临川信任建立（Ch1）

### 场景
M002 宗门广场（谢临川初遇）

### 测试问题
玩家是否在第一印象中认为"谢临川值得信任"？

### 预期玩家反应
- 主动与他完成全部对话（不快速跳过）
- 在后续 Ch3 面临选择时，倾向于相信他

### 验证方式
1. **事件完成度**：M002 谢临川对话事件是否 100% 触发完成（无中途跳过）
2. **行为跟踪**：如果 Ch3 有"相信/怀疑谢临川"的选项，统计选择"相信"的比例

### 通过标准
- ≥70% 的测试者完整看完 M002 对话
- ≥70% 的测试者在 Ch3 选择"相信谢临川"（如果分支存在）

### 技术实现提示
```mz
// 在 M002 事件末尾设置
$gameVariables.setValue(10, $gameVariables.value(10) + 1) // XIE_TRUST_COUNT

// 在 Ch3 选项分支记录
$gameVariables.setValue(11, choiceId) // XIE_CHOICE_AT_CH3
```

## 测试点 TP-02: 家谱调查悬疑感（Ch3）

### 场景
M017 古碑密室（家谱调查事件）

### 测试问题
450 字悬疑文案是否让玩家感到"这东西不对劲"并重复查看？

### 预期玩家反应
- 调查后不立即离开，站在原地
- 重新打开菜单查看物品描述 ≥1 次
- 表现出犹豫或困惑（如反复查看）

### 验证方式
1. **物品使用次数**：家谱物品在获得后 5 分钟内被打开查看的次数
2. **事件停留时间**：M017 调查事件结束后，玩家在相同坐标停留的秒数

### 通过标准
- >60% 的测试者重复查看家谱 ≥2 次
- 平均停留时间 ≥8 秒（读完文案并思考）

### 技术实现提示
```mz
// 物品检查事件中增加
$gameVariables.setValue(20, $gameVariables.value(20) + 1) // REGISTRY_VIEW_COUNT

// 玩家离开调查点前检查停留
if (playerPosition == previousPosition && timeElapsed > 5) {
  $gameVariables.setValue(21, 1) // REGISTRY_STAYED_AFTER
}
```

## 测试点 TP-03: 黑衣人战斗压迫感（Ch3）

### 场景
M020-M023 黑衣人对决（必败战斗）

### 测试问题
玩家是否感受到"必败"的压迫感，并体验谢临川救援的情感冲击？

### 预期玩家反应
- 在败北前尝试 ≥2 种不同战斗策略（攻击/防御/逃跑）
- 败北时产生"绝望感"，而非"没意思"
- 谢临川救援台词后，玩家有情感回应（停留）

### 验证方式
1. **战斗选择多样性**：记录玩家在战斗中的行动类型 count
2. **败北触发时间**：平均第几回合触发 HP≤20%（正常应为 3-5 回合）
3. **事件后停留**：败北后 Scene 3-6 对话前的等待时间

### 通过标准
- >80% 的测试者尝试 ≥2 种不同战斗行动
- 平均败北回合 ≤4（说明黑衣人确实强）
- >50% 的测试者在谢临川台词后停留 ≥5 秒

### 技术实现提示
```mz
// 战斗事件中记录玩家行动
$gameVariables.setValue(30, $gameVariables.value(30) + actionType) // 不同actionType累加

// 败北触发后，统计等待时间
$gameVariables.setValue(31, Graphics.frameCount) // 记录对话开始帧
```

## 测试点 TP-04: 谢临川独白催泪度（Ch3）

### 场景
M025 败逃后谢临川独白（3 页对话）

### 测试问题
280 字独白是否达到"催泪"效果，让玩家感受到"失去"与"代价"？

### 预期玩家反应
- 读完所有 3 页，不快速跳过
- 对话结束后有 ≥10 秒的静止（情感消化）
- 在后续 Ch4 提到谢临川时语气变化

### 验证方式
1. **对话完成度**：是否看完第 3 页
2. **消化时间**：对话结束到下一个事件触发的间隔
3. **后续影响**：Ch4 如果有关谢临川的选项，统计选择"救他/为他报仇"类言论的比例

### 通过标准
- >70% 的测试者完成全部 3 页对话
- 平均消化时间 ≥10 秒
- >60% 的测试者在 Ch4 提及或选择与谢临川相关的选项

### 技术实现提示
```mz
// 每页对话结束设置标志
$gameVariables.setValue(40, 1) // XIE_MONO_P1_READ
$gameVariables.setValue(41, 1) // XIE_MONO_P2_READ
$gameVariables.setValue(42, 1) // XIE_MONO_P3_READ

// 对话结束后启动计时器
$gameTimer.start(10) // 10秒后才允许下一步
```

## 测试点 TP-05: 残我遭遇悬念与二周目意愿（Ch4）

### 场景
M028-M030 深渊镜面空间 + 选择 + 结尾钩子

### 测试问题
- 六残我概念是否清晰？
- 玩家是否想继续二周目？

### 预期玩家反应
- 在绮罗音对话中理解"六人各带一部分"
- 结尾看到二周目按钮后产生"要收集碎片"的冲动
- 在 Ch4 独白中提到"六个"或"全部"

### 验证方式
1. **对话选项**：如果设计"你明白残我机制了吗？"的隐藏选项，统计选择"明白"的比例
2. **二周目按钮点击率**：通关后 5 分钟内点击二周目按钮的比例
3. **独白关键词**：Ch4 结尾独白选项中出现"六个"、"全部"等词的比例

### 通过标准
- >60% 的测试者在独白中提到"六个"或"全部"
- >40% 的测试者在 5 分钟内点击二周目按钮
- >70% 的测试者在选择时表现犹豫（"接受/需要时间/不信"分布均匀）

### 技术实现提示
```mz
// 记录独白选项
$gameVariables.setValue(50, choiceId) // RESIDUE_CHOICE

// 二周目按钮点击
$gameVariables.setValue(51, 1) // NEW_GAME_PLUS_CLICKED

// 关键词检测（简单版：选项ID映射）
if (choiceId == 1) $gameVariables.setValue(52, 1) // 提到"六个"
```

## Playtest 执行流程建议

1. **准备阶段**
   - 确保上述变量已添加到 Database > Variables
   - 为每个测试点添加日志输出（控制台或文件）

2. **测试阶段**（每名测试者）
   - 记录变量初始值 → 完整游戏 → 记录变量最终值
   - 观察员记录：玩家表情、停留时间、重复行为
   - 简短访谈（5分钟）："哪个场景让你印象最深？为什么？"

3. **数据分析**
   - 收集 10-20 名测试者数据
   - 计算各测试点通过率
   - 识别失败点（<60% 通过）的文案或流程问题

4. **迭代优先级**
   - TP-01 失败 → 修改谢临川初遇对白（增加亲和力）
   - TP-02 失败 → 缩短家谱文案或增加更多环境暗示
   - TP-03 失败 → 调整黑衣人战斗数值（降低或增加压迫事件）
   - TP-04 失败 → 精简独白，增加更强的关键词（"代价"、"第一个"）
   - TP-05 失败 → 加强残我概念解释或二周目按钮视觉吸引力
