# WRITING_SKILL_UPGRADE_PATH_v1

目标：系统提升 B 的剧本能力（催泪、反转、悬疑、伏笔、节奏、对白）。
适用：RPG 剧情向项目（Demo -> 正式章回）。

## 1) 学习方向（按优先级）
1. 角色弧线（人物在冲突中变化）
2. 场景目标（每场戏必须推动信息/关系/风险）
3. 伏笔与回收（埋点-误导-揭示）
4. 反转设计（前后逻辑一致，情绪反差）
5. 催泪机制（损失、错过、代价、迟到的真相）

## 3) B 的每周训练任务（固定模板）
文件：`rpg-project/writer-training/week-XX.md`

每周必须提交：
1. 1个“悬疑场景”重写（300~500字）
2. 1个“反转节点”设计（前置伏笔>=3个）
3. 1个“催泪段落”设计（含代价与情绪回收）
4. 1组“短对白优化”对比（原版 vs 精修版）

每项都要附：
- 意图（想让玩家感受什么）
- 技术限制（MZ事件文本长度/触发方式）
- 可执行落点（对应 map/scene/event）

---

## 4) 快速评估标准（监督打分）
每项 0~5 分，总分 25：
1. 信息推进是否清晰
2. 情绪起伏是否有效
3. 伏笔回收是否成立
4. 文案是否可直接落地到事件
5. 是否符合当前 sprint 范围

>=20：可入主线
15~19：可用但需精修
<15：打回重写

---

## 5) 项目内写作红线
- 不允许只“文艺抒情”不推进剧情
- 不允许超长对白堵塞节奏
- 不允许为了反转牺牲逻辑
- 不允许提前透支真相层关键剧透

写作服务可玩体验，而不是脱离系统的纯文本展示。
