# MAP_PRODUCTION_PLAYBOOK_v1

适用对象：A/C 技术节点（RPGMaker MZ）
目标：在不破坏主流程的前提下，快速提升地图观感与可玩性。

## 1) 地图先分区，再摆元素
每张地图先画 4 区：
1. 主路径区（玩家推进）
2. 互动区（NPC/剧情触发）
3. 装饰区（树、石、草、杂物）
4. 阻挡区（墙、栅栏、山体）

规则：
- 主路径必须从入口一眼可识别
- 互动区不能被装饰遮挡
- 阻挡区负责“引导”，不是“封死”

## 2) 三层法（避免平面感）
每张图都要有：
- 背景层：地面变化（草地深浅/石板/泥地）
- 中景层：树木/建筑/岩石（主要结构）
- 前景层：小草/旗帜/碎石/灯笼（细节）

## 3) 动线规则
- 主线路径不超过 2 次明显拐弯就应到关键点
- 每 6~10 秒给玩家一个反馈（调查点/NPC/提示）
- 关键事件前留 2~3 格缓冲区（防止误触）

## 4) Demo-Lite 三图标准
### M001 宗门广场
- 目标：安全感 + 任务引导
- 必备：门楼、中心地标、2~3 栋建筑、石板路
- 事件：入门/谢临川/师尊

### M002 宗门周边
- 目标：过渡探索 + 轻压力
- 必备：弯路、断墙、路标、草药区
- 事件：探索/轻战斗/提示

### M003 古碑区域
- 目标：压迫感 + 冲突高潮
- 必备：古碑大地标、碎石裂痕、冷色调、战斗区
- 事件：古碑调查/黑衣人/必败退场

## 5) 每张图最低验收线
- [ ] 至少 1 个视觉焦点（地标）
- [ ] 至少 3 个可调查点
- [ ] 至少 1 个剧情触发点
- [ ] 路径清晰，不迷路
- [ ] 不影响现有主流程事件触发

## 6) 常见错误
- 只铺地，不立体（看起来像测试图）
- 到处都能走（无引导，玩家发散）
- 事件点被树/建筑挡住
- 关键提示只写一次，玩家错过就卡住

## 7) 输出要求（每次提交）
1. 三图全景截图
2. 主路径标注图
3. 事件触发点位图
4. 回归检查结果（consistency + flow）
