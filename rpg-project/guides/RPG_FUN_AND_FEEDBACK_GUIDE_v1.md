# RPG_FUN_AND_FEEDBACK_GUIDE_v1

适用对象：A/B/C
目标：提升“有游戏感”，不是只完成流程。

## 1) 三种反馈必须存在
1. 即时反馈：玩家一操作就有反应（文本、音效、动画）
2. 进度反馈：玩家知道自己离目标还有多远
3. 情绪反馈：关键节点有氛围变化（音乐/色调/对话强度）

## 2) 5分钟 Demo 的趣味性公式
趣味 = 探索发现 + 明确目标 + 情绪峰值

最低实现：
- 30~45秒出现第一个“可发现点”
- 2~3分钟出现第一个“高压节点”（黑衣人前奏）
- 4~5分钟给“虽败有收获”的强反馈

## 3) 对话趣味规则（B执行）
- 每句尽量短
- 每段对话至少有一个“信息推进点”
- 避免空泛抒情（尤其 Demo 期）
- 关键台词可复述玩家目标

## 4) 事件趣味规则（A/C执行）
- 关键事件前加 1 个预兆
- 关键事件后给 1 个奖励/线索
- 失败不等于白费（必须有新信息）

## 5) 常用“低成本增趣”手段
- 地图路标 + 一句提示
- 调查点里埋 1 行世界观彩蛋
- 关键节点音效变化
- 战前 1 秒停顿 + 暗场
- 道具获得时明确“为什么重要”

## 6) 提交前自检（A/B/C通用）
- [ ] 玩家知道下一步去哪
- [ ] 玩家知道刚刚得到的东西有何意义
- [ ] 关键冲突后有明确推进
- [ ] 5分钟内至少有一次“哇，原来如此”
